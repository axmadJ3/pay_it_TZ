1. Что такое индексы в SQL? Зачем они нужны?  
    Индексы в SQL — это специальные структуры данных, 
    создаваемые для ускорения поиска и выборки данных в таблицах базы данных. 
    Они работают аналогично указателям в книгах, 
    позволяя базе данных быстро находить строки без необходимости сканировать всю таблицу.

    -- Ускоряют выполнение SELECT запросов.
    -- Улучшают производительность поиска (WHERE, ORDER BY, JOIN).
    -- Снижают нагрузку на процессор при работе с большими объемами данных.


2. В чем разница между `INNER JOIN` и `LEFT JOIN`?  
    -- INNER JOIN — возвращает только те строки, 
       у которых есть совпадения в обеих таблицах. 
       Если в одной из таблиц нет соответствующих данных, эта строка не попадет в результат.

       Пример INNER JOIN (возвращает только совпадающие строки)
       SELECT employees.name, departments.department_name
       FROM employees
       INNER JOIN departments ON employees.department_id = departments.id;

    -- LEFT JOIN (или LEFT OUTER JOIN) — возвращает все строки из левой таблицы, 
       даже если в правой таблице нет соответствующих записей. 
       Если соответствий нет, в столбцах правой таблицы будет NULL

       Пример LEFT JOIN (возвращает всех сотрудников, даже если у них нет департамента)
       SELECT employees.name, departments.department_name
       FROM employees
       LEFT JOIN departments ON employees.department_id = departments.id;

    
3. Что такое транзакция? Какие свойства транзакций вы знаете? 
    -- Транзакция — это последовательность SQL-операций, выполняемых как единое целое. 
       Если одна из операций не выполняется успешно, вся транзакция должна быть отменена (ROLLBACK), 
       чтобы избежать неконсистентных данных.

    -- Свойства транзакций (ACID):
      - Atomicity (Атомарность) – либо выполняется вся транзакция, либо не выполняется ничего.
      - Consistency (Согласованность) – данные в БД остаются в корректном состоянии до и после транзакции.
      - Isolation (Изоляция) – параллельные транзакции не должны влиять друг на друга.
      - Durability (Долговечность) – после фиксации (COMMIT) изменения остаются в БД даже при сбоях.
